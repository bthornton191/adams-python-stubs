    values_full ¶
    values_full(*args) -> ValuesView[Object]

    Return all managed objects keyed by their full dot-path name.

    Unlike values, this method uses the full dot-path name (e.g. '.MODEL_1.UDE_INSTANCE_1.PART_1') as the dictionary key, so objects from different UDE instances that share the same short name are all returned — none are silently dropped.

    Prefer this over values whenever the model contains User-Defined Element (UDE) instances whose child objects share identical short names. See ADMS-137078.

    Example: 

    # Instead of (may silently drop duplicates under UDEs):
for part in model.Parts.values():
    ...

# Use (returns every part regardless of name collisions):
for part in model.Parts.values_full():
    ...

    keys_full ¶
    keys_full(*args) -> KeysView[str]

    Return the full dot-path names of all managed objects.

    Unlike keys, this method uses the full dot-path name (e.g. '.MODEL_1.UDE_INSTANCE_1.PART_1') as the key, so objects from different UDE instances that share the same short name are all represented — none are silently dropped.

    Prefer this over keys whenever the model contains User-Defined Element (UDE) instances whose child objects share identical short names. See ADMS-137078.

    items_full ¶
    items_full(*args) -> ItemsView[str, Object]

    Return (full_name, object) pairs for all managed objects.

    Unlike items, this method uses the full dot-path name (e.g. '.MODEL_1.UDE_INSTANCE_1.PART_1') as the key, so objects from different UDE instances that share the same short name are all returned — none are silently dropped.

    Prefer this over items whenever the model contains User-Defined Element (UDE) instances whose child objects share identical short names. See ADMS-137078.

    items ¶
    items() -> ItemsView[str, Object]

    Return (short_name, object) pairs for all managed objects.

    Warning: The dictionary is keyed by short name. If the model contains User-Defined Element (UDE) instances whose child objects share the same short name, only one object per name is returned — later entries silently overwrite earlier ones (ADMS-137078). Use items_full to retrieve all objects by full dot-path name.

    values ¶
    values() -> ValuesView[Object]

    Return all managed objects.

    Warning: Internally keyed by short name. If the model contains User-Defined Element (UDE) instances whose child objects share the same short name, only one object per name is returned — duplicates are silently dropped (ADMS-137078). Use values_full to retrieve all objects regardless of name collisions.

    keys ¶
    keys() -> KeysView[str]

    Return the short names of all managed objects.

    Warning: Returns short names only. If the model contains User-Defined Element (UDE) instances whose child objects share the same short name, only one name per collision is returned (ADMS-137078). Use keys_full to retrieve every object's full dot-path name.
