    Sensor ¶

    Sensor(_DBKey)

    Bases: Object

    Attributes¶

    compare instance-attribute ¶
    compare: Literal['eq', 'ge', 'le']

    Comparison operator used to trigger the sensor: 'eq' (equal), 'ge' (greater than or equal), or 'le' (less than or equal).

    codgen instance-attribute ¶
    codgen: bool

    If True, regenerates a new pivot sequence for matrix factorization when the sensor is triggered.

    halt instance-attribute ¶
    halt: bool

    If True, terminates Adams/Solver execution when the sensor is triggered.

    sensor_print instance-attribute ¶
    sensor_print: bool

    If True, writes to the request, graphic, and results files when the sensor is triggered.

    restart instance-attribute ¶
    restart: bool

    If True, restarts the simulation (reinitializes step size and integration order) when the sensor is triggered.

    sensor_return instance-attribute ¶
    sensor_return: bool

    If True, returns to the command level when the sensor is triggered.

    yydump instance-attribute ¶
    yydump: bool

    If True, dumps the state variable vector when the sensor is triggered.

    bisection instance-attribute ¶
    bisection: bool

    If True, uses a bisection search algorithm to isolate the sensor activation time.

    time_error instance-attribute ¶
    time_error: float

    Temporal tolerance used to isolate the sensor activation time.

    dt instance-attribute ¶
    dt: float

    Redefines the output step interval when the sensor is triggered.

    value instance-attribute ¶
    value: float

    Non-angular value compared against the function expression to determine if the sensor fires.

    error instance-attribute ¶
    error: float

    Allowable error region around value for the trigger comparison.

    angular instance-attribute ¶
    angular: bool

    If True, indicates that the function being tracked is an angular quantity.

    angular_value instance-attribute ¶
    angular_value: float

    Angular value compared against the function expression (degrees).

    angular_error instance-attribute ¶
    angular_error: float

    Allowable error region around angular_value (degrees).

    stepsize instance-attribute ¶
    stepsize: float

    Resets the integration step size when the sensor is triggered.

    function instance-attribute ¶
    function: str

    FUNCTION expression evaluated each time step to determine whether to trigger the sensor.

    user_function instance-attribute ¶
    user_function: List[float | int]
    routine instance-attribute ¶
    routine: str
    evaluate instance-attribute ¶
    evaluate: str

    Expression evaluated and returned when the sensor is triggered.

    user_evaluate instance-attribute ¶
    user_evaluate: float

    Up to 30 values passed to the EVALUATE user subroutine when the sensor triggers.

    evaluate_routine instance-attribute ¶
    evaluate_routine: str

    Name of the user subroutine called for sensor evaluation.

    create ¶
    create(name: str = None, function: str = None, evaluate: str = None, compare: Literal['eq', 'ge', 'le'] = None, value: float = None, error: float = None, angular_value: float = None, angular_error: float = None, time_error: float = None, codgen: bool = None, dt: float = None, halt: bool = None, restart: bool = None, sensor_return: bool = None, yydump: bool = None, sensor_print: bool = None) -> Sensor

    summary

    Parameters:

    Name Type Description Default
    name str

    Name of the sensor

    		 None
    function str

    Specifies a FUNCTION expression to define the sensor.

    		 None
    evaluate str

    Run time function describing the return value

    		 None
    compare str

    Can be "GE", "EQ", or "LE". Specifies what kind of comparison is to be made to initiate the action by the SENSOR.

    		 None
    value float

    Specifies the non-angular VALUE you want to relate to the FUNCTION that Adams is sensing.

    		 None
    error float

    Specifies the absolute non-angular value of allowable error between VALUE and the value of the FUNCTION that Adams is sensing.

    		 None
    angular_value float

    Specifies the angular VALUE you want to relate to the FUNCTION that Adams is sensing.

    		 None
    angular_error float

    Specifies the absolute angular value of allowable error between VALUE and the value of the FUNCTION that Adams is sensing.

    		 None
    time_error float

    Amount of time error allowed for the sensor to trigger

    		 None
    codgen bool

    Specifies that Adams is to generate a new pivot sequence for matrix factorization when the event Adams is sensing has the specified relationship to VALUE.

    		 None
    dt float

    Specifies that the time between consecutive output steps should be redefined. This is done when Adams first senses that the FUNCTION specified has the same relationship as specified to VALUE. Adams uses this value until it is changed.

    		 None
    halt bool

    Specifies that execution should be terminated when the FUNCTION that Adams is sensing, has the specified relationship to VALUE.

    		 None
    restart bool

    Specifies that Adams should restart the integration when the FUNCTION that Adams is sensing has the specified relationship to VALUE. Adams reinitializes the integration step size to HINIT and reduces the integration order to one.

    		 None
    sensor_return bool

    Specifies that Adams should stop the simulation and return to the command level, when the FUNCTION that Adams is sensing has the specified relationship to VALUE.

    		 None
    yydump bool

    Specifies that Adams should dump the state variable vector when the FUNCTION that Adams is sensing has the specified relationship to VALUE.

    		 None
    sensor_print bool

    Specifies that Adams should write data to the request, graphics, and output files when the FUNCTION that Adams is sensing, has the specified relationship to VALUE.

    		 None
